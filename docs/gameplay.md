# Gameplay Design

本文记录 user-web 开始页的四个游戏化 demo。当前目标是验证玩法、交互和 UI 结构，不接入后端、不写入数据库。

## 词汇猎手

顶部掉落中文词汇卡片，玩家移动底部购物车接住词汇。

判定规则：
- 每个掉落词汇带有一个类型。
- 购物车显示当前目标类型。
- 接住词汇时，词汇类型与购物车类型一致视为正确。
- 类型不一致视为错误。

当前 demo：
- 使用固定词汇池。
- 通过左右按钮移动购物车。
- 支持开始、暂停和重置。

## 色彩风暴

玩法与词汇猎手一致，但掉落内容变为颜色文字卡片，购物车目标变为颜色。

判定规则：
- 每张颜色卡片带有颜色标签。
- 购物车显示当前目标颜色。
- 接住的颜色与购物车颜色一致视为正确。
- 颜色不一致视为错误。

当前 demo：
- 使用固定颜色池。
- 卡片视觉颜色与目标颜色一致。
- 复用词汇猎手的接取逻辑。

## 指尖节奏

屏幕分为左右两部分。左侧按照一定节奏自上而下出现中文词汇；全部出现后右侧解封，玩家需要按照同样顺序选择英文翻译卡片。

判定规则：
- 左侧中文词汇按固定顺序出现。
- 右侧英文卡片在左侧序列完成后解锁。
- 玩家每次选择必须匹配当前序号对应的英文翻译。
- 顺序正确则推进，错误则保留当前进度并记错。

当前 demo：
- 使用固定中文-英文词组。
- 支持播放节奏、选择答案和重置。

## 词龙接句

顶部展示一个英文句子，屏幕中散落中文词卡；玩家按住第一个高亮词卡，并依序移动鼠标连接后续词卡，使中文词卡连成一句话。

判定规则：
- 英文句子对应一个中文词序列。
- 第一个待连接词卡高亮显示。
- 只有在当前目标词卡上按下鼠标才会开始链接。
- 按住鼠标移动时，鼠标触碰下一个正确词卡才视为链接成功。
- 仅连接线碰到词卡不算链接。
- 顺序错误视为错误，不推进。
- 松开鼠标会结束本轮链接，已完成的正确进度保留。
- 完成整句中文顺序视为成功。

当前 demo：
- 使用固定句子与中文格子。
- 使用 pointer events 处理按下、移动、松开。
- 使用已连接词卡之间的线条提供视觉反馈，但线条不参与判定。
- 后续可扩展为关卡化句子池、计时和错误回放。

## 风险控制

- 定时器必须在组件卸载或重置时清理，避免资源泄漏。
- 开始、暂停、重置必须避免重复计时器导致竞态。
- 当前 demo 不新增依赖，不修改 API、数据库或迁移。
- 用户可见文案必须走 i18n。
